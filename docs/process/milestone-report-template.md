# Milestone Report Template

Use this concise report format for every milestone close-out.

## 1. Objective
- One to three lines describing milestone intent.

## 2. Verification
- What was verified before implementation.
- File and command evidence.

## 3. Gap Analysis
- What exists.
- What is missing.
- What will be extended versus created.

## 4. Implementation
- Component-by-component summary.
- One commit per component in strict mode.

## 5. Build Result
- Compile/build/test outcomes with command evidence.

## 6. Git Status
- Confirm clean or provide exact remaining deltas.

## 7. Commit / Tag
- Commits created.
- Tags created/updated.

## 8. Next Objective
- Next milestone or next component in sequence.
